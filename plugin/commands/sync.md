# /sync

Explore the codebase to understand its structure, conventions, and patterns.

## Usage

```
/sync
```

## What it does

1. **Explore Structure** - Analyze project layout
2. **Discover Patterns** - Find coding conventions
3. **Build Index** - Create searchable knowledge base
4. **Update Context** - Refresh `.pilot/context.md`

## Why Run /sync

- Helps understand project architecture
- Discovers testing patterns
- Identifies code style conventions
- Updates contextual knowledge

## When to Run

- First time working on a project
- After significant project changes
- When context seems stale

## Output

Updates `.pilot/context.md` with:
- Project structure
- Key files and directories
- Technology stack
- Testing patterns
- Coding conventions

## Related Commands

- `/learn` - Extract reusable knowledge
- `/spec` - Create specification with context
